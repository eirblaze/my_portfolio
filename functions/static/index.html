    種類
    更新案件

    更新前

    • 以前は、htmlによる静的ページを、各国の言語担当とともに更新していた。
    • メールでエクセル表をやりとりしながら翻訳を協議するため、時間がかかる。
    • 中国語と台湾語のページがあるが、それぞれを間違って編集してしまうことがある。
    • サポートが終了したレガシーシステム上で動作していた。
    • ページネーションの機能がないため、将来的に記事が肥大化すると、ページの読み込みが遅くなってしまう。

    更新後

    • CMSにより、htmlのアップロード、リンク作成、メタタグ設定などの手順が不要になり、生産性が向上。
    • 記事は下書きの段階でオンライン共有されるため、メールでのやり取りが不要に。
    • 各言語担当ごとに、記事の編集権限を限定。中国語と台湾語を取り違えることはなくなった。
    • レガシーシステムを最新のものに置き換え。伴って、既存のphpを書き換えた。
    • セキュリティを高めるWordPressプラグインを導入。
    • ajaxと呼ばれる、非同期読み込みの仕組みを導入することで、以下を実現。
      • 「続きを表示するボタン」により、ページネーションを不要にした。
      • 記事のカテゴリを出しわける際も、ページ遷移が不要になり、表示が高速化された。
    • その他、SEO対策のための仕組みを新設
